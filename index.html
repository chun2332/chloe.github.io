使用 React Hook 更新、紀錄會員的資料編輯狀態

Header 根據 JWT 偵測會員編號顯示歡迎訊息

密碼修改考量帳戶安全，沒有紀錄會員的舊密碼，兩次輸入結果必須相同，才能完成修改

每筆訂單皆根據會員的消費紀錄去做 API 串接顯示，並且每筆訂單都可以根據訂單狀態的差異，透過 Post Request 改變訂單狀態；分類拉表的部分，藉由 React Hook 減少過多 Reload 造成會員體驗不便，訂單根據狀態分頁顯示，提升會員使用效能

串接 API 準確撈出根據會員消費的訂單記錄呈現訂單細節各項資訊，點擊前往評價，可連結至商品細節頁，供會員作商品評價或查看商品細節

網站根據節慶，不定時發放優惠券予會員，會員複製成功後（複製完成系統會貼心提醒會員複製成功），會員可至購物車結帳時做使用

票卷顯示係根據會員的票卷訂單紀錄，利用 Array map 產生每筆票卷資訊，點選活動細節，可連結至活動細節頁，供會員做更詳細地查看

根據會員的收藏行為利用 Node.js、MySQL 與資料庫做 API 串接，點選愛心 icons 即可移除收藏，點擊購物車 icons 可連結至商品詳細頁，讓會員進行更近一步的選購
